# Observer Pattern

> Source(s): raw/ch-03-Continuous Integration and Delivery

观察者模式定义了一种一对多依赖关系：当一个对象（被观察者）状态发生变化时，所有依赖于它的对象（观察者）都会自动收到通知。该模式是事件驱动架构、响应式编程框架和发布-订阅消息系统的核心基础。

在软件设计中，观察者模式最早由 GoF 四剑客编目，至今仍广泛应用于各类系统。在 JavaScript 中，通过事件监听（Event Listeners）和 RxJS 的 Observable 模式实现；在后端系统中，消息队列如 RabbitMQ、Kafka 则大规模实现了发布-订阅模式。观察者模式的本质是解耦——被观察者无需知道观察者的具体实现细节，只需维护一个订阅列表，并在状态变更时广播通知。

该模式常用于以下场景：
- UI 框架中的状态变化通知（如视图随模型更新）
- 异步事件处理系统（如鼠标点击、网络请求完成）
- 分布式系统中的消息发布与订阅（如事件总线、消息队列）
- 响应式编程中的数据流传播（如 RxJS、ReactiveX）

值得注意的是，观察者模式与[策略模式](??)和[工厂模式](??)等经典模式一样，为开发者提供了可复用的架构解耦方案。在现代化软件开发中，它也是[持续集成](??)和[管道即代码](??)实践中处理构建状态通知的常见模式（例如 CI 流水线失败时自动通知团队成员）。
